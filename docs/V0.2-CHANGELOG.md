# Consite v0.2 — What's New

> While you were demoing v0.1, here's what got built.

## Three new magic moments

### 1. AI Daily Log (`/daily-log`)

The supervisor's end-of-shift wow moment.

- Tap one button → AI synthesizes the day's clock-ins, FLHA submissions, weather, and incidents into a written daily log.
- Plain professional English. Past tense. Factual.
- Editable in-place before saving.
- Saves to `/projects/{pid}/dailyLogs/{date}`.

**Time saved:** ~30 min per super per day. Replaces a paper site diary that 70% of small contractors keep poorly or not at all.

**Demo line:** "Watch this — at the end of a shift, instead of writing your daily log from scratch, the app already drafted it from everything that happened today."

### 2. Photo + voice → Deficiency (`/deficiency`)

Worker sees a problem on site. Snaps a photo. Says one sentence.

- AI writes a structured deficiency report: title, description, trade, severity, recommended action.
- Goes straight into the punch list.
- BC-aware: severity tiers map to industry norms.

**Replaces:** iPhone Notes app + WhatsApp + memory.

**Demo line:** "Imagine your foreman walks the site at 4pm. Every issue he spots, he just takes a photo and talks for 3 seconds. By 4:30 you have a complete punch list in your inbox."

### 3. Receipt scanner (`/receipt`)

Photo of a Home Depot, Lowe's, Dick's Lumber receipt → structured job-cost line item.

- Vendor, date, total, GST/PST split (BC-aware tax handling).
- Category auto-detected (lumber, fasteners, tools, etc.).
- Line items extracted.
- Saves to `/projects/{pid}/expenses/{id}`.

**Replaces:** shoebox of receipts and the foreman's weekly spreadsheet.

**Demo line:** "Every receipt your foreman throws on the truck dashboard becomes a job-cost line item. Photo. Done. No more pleading for them at month-end."

## New screens

| Path | Description |
|---|---|
| `/(worker)/daily-log` | AI-generated daily log |
| `/(worker)/deficiency` | Report a deficiency with photo + voice |
| `/(worker)/receipt` | Scan a receipt → job cost |
| `/(worker)/punch-list` | View all deficiencies, filter by status/severity |
| `/(worker)/certifications` | Worker's own tickets (WHMIS, fall arrest, first aid, etc.) |
| `/(admin)/compliance` | Admin overview of expiring certs across all workers |

## Data model additions

```
/projects/{pid}/deficiencies/{id}
  { title, description, trade, severity, recommendedAction,
    status, photoUri, voiceTranscript, reportedBy, reportedAt }

/projects/{pid}/expenses/{id}
  { vendor, date, subtotalCents, gstCents, pstCents, totalCents,
    category, lineItems[], photoUri, costCode, recordedBy }

/projects/{pid}/dailyLogs/{YYYY-MM-DD}
  { date, log, summary, flagged[], submittedBy, aiGenerated }

/users/{uid}/certifications/{certId}
  { type, displayName, issuer, certificateNumber,
    issuedAt, expiresAt, documentStoragePath }
```

## Form quality of life

- **Per-field AI highlighting.** Fields that AI pre-filled now show a subtle teal left-edge accent. As soon as the worker edits an AI-filled field, the accent disappears (it's their value now, not AI's).
- **"AI auto-filled" banner improvements.** Now shows a confidence chip (`high` / `medium` / `low`) so workers know how much to trust the pre-fills.

## Worker dashboard updates

The default dashboard now surfaces all v0.2 features in this order:

1. Clock In / Out
2. FLHA Forms (AI auto-filled)
3. Report Issue (photo + voice)
4. Scan Receipt (to job cost)
5. Daily Log (AI-written)
6. Punch List (open issues)
7. My Tickets (WHMIS, fall arrest)
8. Projects

The merge logic was rewritten: defaults + Firestore overrides now combine. So you can still customize the dashboard from Firestore *and* new local features automatically appear.

## Updated Firestore rules

New `match` blocks added for `deficiencies`, `expenses`, `dailyLogs`, and `users/{uid}/certifications`. Workers create their own records; admins manage everything; supervisors approve/resolve.

**Action required:** redeploy rules.

```bash
firebase deploy --only firestore:rules
```

## Admin compliance overview

New screen at `/(admin)/compliance`. Aggregates certifications across all workers using a Firestore `collectionGroup` query. Shows:

- Count of expired / expiring soon / valid
- Sorted list of urgent items with worker name + days remaining

**Demo this to office managers** — it's their pain killer.

**v0.3 will add:** automated email/SMS reminders, one-click audit pack export, and a `collectionGroup` index (Firestore will prompt for it the first time the screen loads — click the URL to create it).

## What stayed the same

- Authentication, GPS clock-in, FLHA form flow — all the v0.1 stuff still works exactly as before. Nothing was broken or removed.
- The Cloud Functions backend was already deployed in v0.1 (`aiFillForm`, `aiExtractHazards`, `aiAnalyzeDeficiency`, `aiAnalyzeReceipt`, `aiGenerateDailyLog`). v0.2 just adds the screens that call them.

## Cost impact

Each new AI feature uses the same `aiAnalyze*` / `aiGenerate*` Cloud Functions that already exist:

- Deficiency analysis: ~$0.005 each (vision call on Haiku)
- Receipt OCR: ~$0.005 each (vision call on Haiku)
- Daily log: ~$0.02 each (longer synthesis on Sonnet)

For a 50-worker company with one super running 5 deficiency reports, 2 receipts, and 1 daily log per day across 3 active projects: about **$1.50/day** in Anthropic costs. Trivial vs the value.

## What's NOT in v0.2 (deliberately deferred)

- **Offline submission queue** — workers without signal still see "Cannot submit" errors. Pushed to v0.3.
- **Punjabi UI** — needs every label translated + a language switcher. Pushed to v0.3.
- **Admin form builder UI** — schemas still managed by manually editing Firestore. v0.3.
- **Cert photo upload + AI extraction** — workers still type their cert details. Photo upload comes with v0.3 Storage integration.
- **Plan markup on deficiencies** — deficiencies live as a list, not as pins on a drawing. v0.4.
- **Two-week look-ahead** — distinct planning feature. v0.3.
- **QuickBooks export** — expenses save with cost codes, but export to QBO is v0.4.

## Try the new flows

1. From the worker dashboard, tap **Report Issue**
2. Take a photo (or pick one from your library)
3. Hold the mic and say something like "drywall water damage in the corner, needs replacement"
4. Watch the deficiency report write itself
5. Tap "Add to punch list"
6. Go back, tap **Punch List**
7. Your deficiency appears in the list

Repeat with Receipt scanner (photo of any receipt) and Daily Log (just tap Generate — even if it's mostly empty for testing, you'll see how it works).
